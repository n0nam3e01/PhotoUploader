# Мемориальная страница — сбор фотографий

Спокойный одностраничный сайт памяти. Посетители загружают фотографии, на которых
они запечатлены вместе с человеком, которого поминают. Фото уходят в указанную
папку Google Drive. Интерфейс на двух языках: русский (по умолчанию) и казахский (ҚАЗ).

## Файлы

| Файл | Назначение |
|------|------------|
| `index.html` | Разметка страницы |
| `styles.css` | Оформление (спокойная палитра, адаптивно, mobile-first) |
| `script.js` | Логика, словарь i18n и **все настройки сверху файла** |
| `apps-script/Code.gs` | Код для Google Apps Script (приём фото в Google Drive) |

## Что нужно заполнить (быстрый список)

1. **`script.js` → `PERSON`** — заголовок (имя/название) и путь к картинке.
2. **`script.js` → `UPLOAD_ENDPOINT`** — URL веб-приложения Apps Script (см. ниже).
3. **`apps-script/Code.gs` → `FOLDER_ID`** — ID папки Google Drive.
4. (необязательно) тексты в словаре `I18N` и контакт в подвале (`footerContact`).

---

## Шаг 1. Папка в Google Drive и её ID

1. Создайте папку в [Google Drive](https://drive.google.com), например «Фотографии памяти».
2. Откройте её. Посмотрите на адрес в браузере:
   `https://drive.google.com/drive/folders/`**`1AbCdEfGhIjKlMnOpQrStUvWxYz`**
3. Выделенная часть после `folders/` — это и есть **FOLDER_ID**.
4. Вставьте его в `apps-script/Code.gs`:
   ```js
   var FOLDER_ID = '1AbCdEfGhIjKlMnOpQrStUvWxYz';
   ```

## Шаг 2. Развёртывание Apps Script (получаем UPLOAD_ENDPOINT)

1. Откройте [script.google.com](https://script.google.com) → **Новый проект**.
2. Удалите содержимое `Code.gs` и вставьте код из `apps-script/Code.gs`.
   Не забудьте подставить `FOLDER_ID` (из шага 1).
3. Нажмите **Развернуть → Новое развёртывание** (Deploy → New deployment).
4. Тип развёртывания (шестерёнка) → **Веб-приложение** (Web app).
5. Настройки:
   - **Запуск от имени** (Execute as): *Я* (Me / ваш аккаунт).
   - **У кого есть доступ** (Who has access): **Все** (Anyone).
     > Это нужно, чтобы сайт мог отправлять фото без входа в Google.
6. Нажмите **Развернуть**. При первом запуске Google попросит выдать разрешения —
   согласитесь (выберите аккаунт → «Дополнительно» → «Перейти к проекту» → «Разрешить»).
7. Скопируйте **URL веб-приложения** — он выглядит так:
   `https://script.google.com/macros/s/AKfyc.../exec`
8. Вставьте его в `script.js`:
   ```js
   const UPLOAD_ENDPOINT = 'https://script.google.com/macros/s/AKfyc.../exec';
   ```

> **Проверка:** откройте URL в браузере — должно появиться
> `{"status":"ok","message":"Memorial upload endpoint is running."}`.

> **Важно:** после изменения кода в Apps Script делайте
> **Развернуть → Управление развёртываниями → ✎ Изменить → Новая версия**,
> иначе старая версия продолжит работать.

## Шаг 3. Хостинг самого сайта

Сайт — это статика (HTML/CSS/JS), ему не нужен сервер. Любой вариант подойдёт:

- **GitHub Pages** — залейте файлы в репозиторий, включите Pages в Settings → Pages.
- **Netlify / Cloudflare Pages** — перетащите папку в дашборд (drag-and-drop деплой).
- **Vercel** — `Import Project` из репозитория.
- **Свой хостинг** — просто загрузите `index.html`, `styles.css`, `script.js`
  (и портрет, если есть) в одну папку.

Локально для проверки можно просто открыть `index.html` в браузере, но загрузка фото
заработает только при настроенном `UPLOAD_ENDPOINT`.

## Как это работает

- Фото отправляется по одному POST-запросом на Apps Script в виде JSON
  `{ filename, mimeType, data(base64) }`.
- Запрос отправляется с заголовком `Content-Type: text/plain`, чтобы браузер не делал
  предварительный CORS-запрос `OPTIONS` (Apps Script его не обрабатывает).
- Apps Script декодирует base64 и кладёт файл в папку Drive с меткой времени в имени.

## Ограничения и настройки

- Разрешённые форматы: **JPG, PNG, HEIC, WEBP**. Меняются в `script.js`
  (`ALLOWED_TYPES` / `ALLOWED_EXTENSIONS`).
- Лимит размера файла: **15 МБ** (`MAX_FILE_SIZE_MB` в `script.js`).
- HEIC браузеры не показывают в превью — выводится карточка с именем файла,
  но сам файл загрузится нормально.

## Возможные проблемы

| Симптом | Причина / решение |
|---------|-------------------|
| «Загрузка пока не настроена» | Не заменён `UPLOAD_ENDPOINT` в `script.js`. |
| Ошибка сети при отправке | Проверьте URL и что доступ к веб-приложению = **Все (Anyone)**. |
| Файлы не появляются в Drive | Неверный `FOLDER_ID` или не сделано **новое развёртывание** после правок. |
| Изменили `.gs`, ничего не поменялось | Нужно развернуть **новую версию** (см. примечание в шаге 2). |
